## Key Conventions

All documentation follows these standards:
- Spanish for business logic and technical concepts
- English for infrastructure and platform-specific terms
- No emojis - professional tone throughout
- Clear hierarchy with heading levels (H1 for page title, H2 for sections)
- Code examples with syntax highlighting

## Contributing to Documentation

When adding or updating docs:
1. Ensure technical accuracy
2. Keep language clear and concise
3. Update table of contents in relevant README files
4. Link related documents for navigation
5. Include code examples where helpful

For detailed contribution guidelines, see [Contributing Guide](guides/CONTRIBUTING.md).
